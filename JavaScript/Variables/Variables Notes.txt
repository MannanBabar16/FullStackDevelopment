1. Variables in JavaScript

A variable is a named reference to a value stored in memory.

JavaScript does not store values in variables.
It stores references to values.

This distinction will matter later (especially with objects).

--------------------------------------------------------------------------------------------------

var (Legacy ‚Äì understand, don‚Äôt use)

Characteristics:
Function-scoped (NOT block-scoped)
Hoisted (initialized as undefined)
Can be redeclared
Can be reassigned

var a = 10;
var a = 20; // allowed (dangerous)

Hoisting behavior
console.log(x); // undefined
var x = 5;

What actually happens:
var x;
console.log(x);
x = 5;

Why var is bad

Causes accidental overwrites
Breaks block logic
Makes debugging harder

‚úÖ Rule:
You should know var for legacy code and interviews.
‚ùå You should not use it in modern code.

--------------------------------------------------------------------------------------------------

let (Modern, default choice)

Characteristics:
Block-scoped
Hoisted but in Temporal Dead Zone (TDZ)
Cannot be redeclared
Can be reassigned

let x = 10;
x = 20; // allowed


Block scope:
{
  let a = 5;
}
console.log(a); // ReferenceError


Temporal Dead Zone
console.log(b); // ReferenceError
let b = 10;

TDZ exists to prevent using variables before they‚Äôre logically initialized.

3Ô∏è‚É£ Objects (Non-Primitive)
Objects are collections of key-value pairs.

let user = {
  name: "Alex",
  age: 25
};

Object Characteristics:
Stored by reference
Mutable
Keys are strings or symbols

Reference behavior (CRITICAL):
let obj1 = { a: 1 };
let obj2 = obj1;

obj2.a = 100;
console.log(obj1.a); // 100

Both variables point to the same memory location.

This single concept will:
Save you from bugs
Help you understand React, state, Redux, backend payloads

Comparing objects:
{} === {} // false

Why?
Different memory references

--------------------------------------------------------------------------------------------------

Primitives are immutable in JavaScript because they are stored and accessed by value, and any operation that seems to modify them actually creates a new value in memory.
Objects are mutable because variables hold references to them, and their internal properties can be changed without changing the reference.

const prevents reassignment of the variable, not mutation of the object it references.
Arrays and objects are mutable, so their contents can change as long as the reference stays the same.
